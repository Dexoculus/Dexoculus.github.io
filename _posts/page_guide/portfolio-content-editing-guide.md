## Purpose

This note documents the editing surface of this portfolio site. The goal is to keep routine content updates separate from Astro rendering logic, so profile, CV, project, note, media, and homepage edits can be made directly.

## Edit Map

| Content | Edit here |
| --- | --- |
| Name, role, summary, education, experience, awards, skills, links | `src/data/profile.ts` |
| Home page front text, buttons, site index labels | `src/data/home.ts` |
| Projects | `_projects/**/*.md` |
| Notes | `_posts/**/*.md` |
| Images, videos, PDFs, downloads | `public/assets/` |
| CV PDF download | `public/assets/CV.pdf` |
| Plot and chart usage | `CHARTS.md` and `Plot and Colorbar Authoring Guide` |
| Visual design reference | `guideline.html` |

Do not edit `dist/`. It is generated by the Astro build.

## Profile And CV

Most CV content is centralized in `src/data/profile.ts`.

Use this file for:

- `name`, `handle`, `role`, `location`, and `email`
- `brandLine`, which controls the small line under the name in the top-left navigation brand
- profile summary and research interests
- skill groups
- education
- experience
- publications
- presentations
- awards
- external links such as GitHub, ORCID, PyPI, and Google Scholar

The `/about/` page reads this file directly. For example, updating an internship period or adding an award should usually be done in `profile.experience` or `profile.awards`.

## Home Page Copy

The landing page copy is centralized in `src/data/home.ts`.

Common edits:

```ts
front: {
  image: "/assets/front.webp",
  role: "Robotics, computer vision, physical AI, and data science.",
  summaryPrefix: "A working archive of selected systems..."
}
```

Project and note counts are automatic. They come from the actual Markdown files under `_projects` and `_posts`.

## Projects

Project files live under `_projects/`. Subfolders are allowed:

```text
_projects/
  robotics/
    imitation-learning-pick-place.md
  vision/
    anomaly-detection.md
```

Recommended project frontmatter:

```md
---
name: Imitation Learning for Pick and Place Tasks
description: End-to-end imitation learning pipeline for industrial manipulation.
tools: [LeRobot, ACT, Diffusion Policy, Docker]
featured: true
featured_order: 1
image: /assets/projects/pick-place.webp
external_url: https://example.com
---
```

Use `video` when the project should show a video preview:

```md
video: /assets/video/pick-place.webm
```

Use globally unique filenames. The generated project URL is based on the project name, so similar titles can collide.

## Notes

Note files live under `_posts/`. Subfolders are allowed:

```text
_posts/
  robotics/
    ai-worker-policy-guide.md
  linux/
    cuda-device-count-case-report.md
```

Recommended note frontmatter:

```md
---
title: AI Worker Policy Deployment Guide
description: A practical guide to deploying a trained policy on the ROBOTIS AI Worker platform.
tags: [ROBOTIS, AI Worker, Physical AI Tools, Robotics]
featured: true
featured_order: 1
image: /assets/notes/ai-worker-policy.webp
---
```

The current design system does not require `date`, `style`, or `color` frontmatter.

## Featured Home Items

To show a project or note on the home page:

```md
featured: true
featured_order: 1
```

Lower `featured_order` values appear earlier.

If a file should remain only on the full `/projects/` or `/note/` index, omit `featured` or set it to `false`.

## Media

Put static assets in `public/assets/`.

Use absolute public paths in Markdown:

```md
![Training UI](/assets/notes/training-ui.webp)

<video src="/assets/video/pick-place.webm" controls muted playsinline></video>
```

For card previews, the site can use:

- `image` frontmatter
- `video` frontmatter
- the first Markdown image in a note
- the first HTML image or video tag in a note
- the first local video link in a note

For performance, prefer `.webp` for images and `.webm` or optimized `.mp4` for videos.

## Charts And Plots

Use fenced `chart` or `plot` blocks inside notes when a post needs bar plots, box plots, heatmaps, function plots, or lightweight interactive parameters.

The complete chart syntax is documented in `CHARTS.md` and in the note `Plot and Colorbar Authoring Guide`.

## Local Build Check

Run a build before pushing:

```powershell
npm.cmd run build
```

For local preview:

```powershell
npm.cmd run dev
```
