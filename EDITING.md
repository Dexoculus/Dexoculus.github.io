# Content Editing Guide

This site is structured so common portfolio edits can be made without touching the rendering logic.

The same guide is also published inside the site at `/note/portfolio-content-editing-guide/`.

## Fast Edit Map

| What to edit | File or folder |
| --- | --- |
| Name, profile summary, education, experience, awards, skills, links | `src/data/profile.ts` |
| Home page front text, buttons, site index labels | `src/data/home.ts` |
| Projects | `_projects/**/*.md` |
| Notes | `_posts/**/*.md` |
| Images, videos, PDFs, downloadable files | `public/assets/` |
| CV PDF download | `public/assets/CV.pdf` |
| Chart and plot usage guide | `CHARTS.md` |
| Visual design reference | `guideline.html` |

Do not edit `dist/`. It is generated by the build.

## Home Page

Edit `src/data/home.ts` for the main front page copy.

Common fields:

```ts
front: {
  image: "/assets/front.webp",
  role: "Robotics, computer vision, physical AI, and data science.",
  summaryPrefix: "A working archive ..."
}
```

The project and note counts are automatic. They are calculated from `_projects` and `_posts`.

## CV And Profile

Edit `src/data/profile.ts`.

Use this file for:

- `name`, `handle`, `role`, `location`, `email`
- `brandLine`, which controls the small line in the top-left navigation brand
- `summary`
- `researchInterests`
- `skillGroups`
- `education`
- `experience`
- `publications`
- `presentations`
- `awards`
- profile links such as GitHub, ORCID, PyPI, and Scholar

The `/about/` page reads this data directly.

## Projects

Add project Markdown files under `_projects/`. Subfolders are allowed:

```text
_projects/
  robotics/
    imitation-learning-pick-place.md
  vision/
    anomaly-detection.md
```

Recommended project frontmatter:

```md
---
name: Imitation Learning for Pick and Place Tasks
description: End-to-end imitation learning pipeline for industrial manipulation.
tools: [LeRobot, ACT, Diffusion Policy, Docker]
featured: true
featured_order: 1
image: /assets/projects/pick-place.webp
external_url: https://example.com
---
```

Use `video` instead of `image` for a video thumbnail:

```md
video: /assets/video/demo.webm
```

Projects are sorted by the leading sequence number in the filename when present, for example `(11) pick-place.md`.

## Notes

Add notes under `_posts/`. Subfolders are allowed:

```text
_posts/
  robotics/
    ai-worker-policy-guide.md
  linux/
    cuda-device-count-case-report.md
```

Recommended note frontmatter:

```md
---
title: AI Worker Policy Deployment Guide
description: A practical guide to deploying a trained policy on the ROBOTIS AI Worker platform.
tags: [ROBOTIS, AI Worker, Physical AI Tools, Robotics]
featured: true
featured_order: 1
image: /assets/notes/ai-worker-policy.webp
---
```

You do not need `date`, `style`, or `color` frontmatter. The current design system ignores those fields.

If a note contains a Markdown image, HTML image, video tag, or local video link, the note card can automatically use the first detected media item as its thumbnail.

## Featured Home Items

To show a project or note on the home page:

```md
featured: true
featured_order: 1
```

Lower `featured_order` values appear earlier.

## Assets

Put static files in `public/assets/`.

Reference them from Markdown like this:

```md
![Training UI](/assets/notes/training-ui.webp)

<video src="/assets/video/pick-place.webm" controls muted playsinline></video>
```

For performance, prefer:

- `.webp` for images and animated visual snippets
- `.webm` or optimized `.mp4` for videos
- short muted preview videos for cards

## Build Check

Run this before pushing:

```powershell
npm.cmd run build
```

For local preview:

```powershell
npm.cmd run dev
```
